HTML Data Definition and Display

The HTML data display is written in the div block or any perfect alternative. The classname given to the parent element of the navigation data would programatically be transversed from the `data-target` attribute from the navigation button. The `data-target` should point to the `id` of the main menu.
The navigation consists of the hash navigation which supports the device back button. Every other module in the navigation program partially depends on the hash navigation module. The navigation drawer module requires a button as an alternative for conditions where the drawer event may not work.
The navigation menu, for responsive designs, needs two attributes, the max-width the hash nav should function with and when the drawer should function with. The navigation drawer mainly targets touch devices and not desktop devices. By default and standard it is 767px. This doesn't support both tablet and desktop. On tablet and desktop the navigation menu isn't hidden, therefore there is no use. 
<div class="container">
  <div class="navbar affix top">
    <a href="#menu" class="nav-menu-opener material-icons" data-target="navigation">menu</a>
  </div>
  <div class="backdrop"></div>
  <div class="main-menu left" id="navigation" data-hash-nav-max-width="767" data-nav-drawer-max-width="767">
    <ul class="override">
      <li>
        <a href="examples.html" class="nav-link">Examples</a>
      </li>
    </ul>
  </div>
</div>

Script Initialization

The script handler for the navigation is initialised right from onload. Every plugin data of the navigation is as an extension of the jQuery object prototype as a lot of sites make use of jQuery, and to reduce complexity we consider it more convinient to build it on top of the jQuery library. A compulsory argument at index 0 of value "init" is passed to the nav method and at index 1 of the argument array is the configuration object for the navigation.
Note: If you do not provide the configuration object, it would be gussed by Navigatio.fn.defaultConfig and may not be correctly guessed. It is better you provide the configurationObject as the second parameter.
<script>
  $("nav-menu-opener").nav("init", {
    direction: "left",
    transition: 400,
    event: "click",
    backdrop: true
  });
</script>

CSS Classes and Styles

We provide a css file as the basic template for the navigation plugin. It contains style definitions for the navigation data. The style definitons are the default and can be tweaked to make it fit your designs. Some examples and user's guide are given below.

CSS Class Guides

The .navbar is given to the navbar; a class, .affix gives the navbar a fixed position; the .top is used to fix the navbar to the top. Every link in the navbar and menu can be given the .nav-link class, and the .material-icons, .nav-menu-opener for the button that opens the menu. 
.backdrop {
 background: rgba(0,0,0,.7);
}
.main-menu.offset {
 ...
}
The .backdrop class provides a back overlay for the menu. It is not totally opaque. It is fixed and has both height and width of 100%. Furthermore, the .backdrop.offset: this is used with the .main-menu.offset class. This means the menu is not attached to the top and so also mustn't the backdrop.
Add a .head .lead class to a div in the .main-menu class to add a header to the navigation menu. You may also provide a back button to easily close nav.
